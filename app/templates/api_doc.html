Overview

This document focuses on the WebSocket API for AI Coach realtime chat (token + sentence streaming, optional TTS+visemes with timing, slide updates). For the REST API, see API Docs.

ðŸ”‘ Requirements
  • Api-Key (tenant-scoped)
  • bot_id (UUID of the Agent)
  • thread_id shaped as ^user_[0-9a-f]{16}$ (example: user_ab12cd34ef56gh78)
  • Origin/Host/IP must be allow-listed for the API key

WebSocket Endpoint

wss://ai-coach-pmkn.onrender.com/ws/chat/?bot_id=<uuid>&thread_id=<user_16hex>&website_language=en

Authentication & Allow-list

  • Provide the API key via one of:
    • Header: Authorization: Api-Key <YOUR_API_KEY>
    • Query param: ?api_key=<YOUR_API_KEY>
    • Cookie: api_key=<YOUR_API_KEY>
  • Additionally, at least one of the allow-list checks must pass: Origin/Referer host, Host header, or client IP/CIDR.

Limits & Behavior

  • Inbound message rate: up to 20 Hz (excess is ignored).
  • Audio upload size: up to 25 MiB.
  • Audio formats: auto-detected from data URL or hint: webm, wav, m4a, ogg, mp3.
  • STT model: OPENAI_STT_MODEL env (default whisper-1).
  • Thread safety: one active run at a time; new queries cancel the previous run.

Audio + viseme streaming

The server sends TTS audio with synchronized visemes. Visemes are 15-length frames in either ARKit or OVR layout.

Field Type Notes
audio string Data URL (data:audio/<mime>;base64,â€¦) or base64
viseme array OR object Either raw frames number[ ][15], or
{ frames:number[ ][15], times?:number[ ], fps?:number }
viseme_format string "arkit15" (default) or "ovr15"
viseme_times number[ ] (sec) Optional. Per-frame timestamps; preferred for perfect sync.
viseme_fps number Optional. Used to derive times if viseme_times are absent.
local_time string Client-local echo, informative only.

Python (websockets)

import asyncio, json, websockets

API = "<YOUR_API_KEY>"
BOT = "6a97a3c4-5d25-47a8-bb31-b6f35c98e6b7"
THR = "user_ab12cd34ef56gh78"

async def main():
    url = f"wss://ai-coach-pmkn.onrender.com/ws/chat/?bot_id={BOT}&thread_id={THR}&website_language=en"
    async with websockets.connect(url, extra_headers={"Authorization": f"Api-Key {API}"}) as ws:
        await ws.send(json.dumps({"type":"text_query","text":"Plan my week","local_time":"2025-11-05 10:00:00","muteAudio":True}))
        async for raw in ws:
            msg = json.loads(raw)
            print(msg)

asyncio.run(main())

Audio Payload Tips

  • Send data:audio/<mime>;base64,.... (recommended) or plain base64 + format hint.
  • Server infers extension from data URL or format (wav/m4a/ogg/webm/mp3).
  • For lip-sync quality, prefer viseme_times (seconds). If unavailable, provide viseme_fps. Clients should interpolate between frames.
  • On playback, queue chunks and honor mute_audio/stop_audio immediately.
